# 규칙
- 아래 source에 **실제로 나타난 내용만** 사용한다. source에 없는 사실/수치/아키텍처를 지어내지 않는다.
- 확인되지 않은 것은 "확인 필요" 같은 표시를 남기고 추측으로 단정하지 않는다.
- 구조는 **문제 → 원인 → 해결 → 결과 → 배운 점** 흐름을 기본으로 한다.
- 긴 도입부 없이 바로 문제 상황으로 진입한다.
- 과장·홍보성 문체, 자기계발 에세이 톤, 일반론을 피한다.
- 코드 블록은 source에 근거가 있을 때만, 짧게 사용한다.
- source에 이미지 마크다운(`![설명](url)`)이 있으면 관련 위치에 그대로 가져다 쓸 수 있다. 단 존재하지 않는 이미지를 새로 만들지 않는다.
- 한국어, 기술 블로그 톤으로 작성한다.
